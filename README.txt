Task 5 - Web Application Deployment and Cloud Hosting
1. Project Objective
To transition a local Student Management System to a live production environment, ensuring accessibility
 and stable database management.

2. System Architecture

Framework: Flask (Python)

Database: SQLite3

Server: Gunicorn

Hosting: Cloud PaaS

3. Implementation Steps

Configuration: Setup secure session keys for user authentication.

Database Fix: Automated table creation for cloud-side data handling.

Optimization: Streamlined dependencies for efficient cloud performance.

Deployment: Integrated GitHub with a production server for live routing.

4. Core Functionalities

Authentication: Secure admin login and registration gateway.

Dashboard: Protected interface for database management.

CRUD Operations: Ability to Create, Read, Update, and Delete student records live.

Session Management: Encrypted user state tracking across the web service.

5. Conclusion
The deployment successfully confirms a functional full-stack lifecycle,
 moving from local development to a stable cloud-hosted application.
